# Smart_Account
for a user to interact with a platform that uses account abstraction, you need first have a list of details and transaction to want to do in a userOp. Now we are focusing on creating the userOp with the provided contracts. The important fields in a userOp are: sender, nonce, initCode, callData, signature, payMasterandData. In this repo we will focus on the fileds besides paymasterandData. to get the sender we deploy accountFactory that has a function called createAccount that returns a transaction inside of an address we use staticCall to get the sender address and to verify that we have the correct address we get the callData of createAccount then add it with the accountFactoryAddress and rempving the "0x" from the craeteAccount of cause for it to be come a valid hash. then we get the callData buy calling the execute function in an encodedFunction in Account factory and then and we then deploy EntryPoint to verify that we got the correct sender.(We actual started with the EntryPoint one before using staticCall actually). We then sign the useOpHash, why do we sign the userOpHash and not the userOp, before that we just have a place holder for the signature. Then for the callData part we decode the callData for execute and for executeBatch(pointer, value, function selector)
